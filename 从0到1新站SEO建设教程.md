# 从0到1新站 SEO 建设教程

使用 SuperSEO + SEO Machine + Claude SEO 三件套，从零开始建设一个新网站的 SEO。

---

## 新站的独特挑战

从0到1的网站和老站改造完全不同：

- **没有域名权重**：Google 不认识你，爬取频率低，索引慢
- **没有内容资产**：零页面、零内链、零话题覆盖
- **没有外链基础**：没人引用你，没人信任你
- **没有历史数据**：GSC 里没数据，不知道什么词能带来流量

所以策略也不同——不能一上来就铺大量内容，不能一上来就追高难度词。**核心原则：先聚焦窄话题建立权威，再逐步扩张。**

---

## 第〇步：一次性初始化（所有工具）

新项目到手，先把三个工具的配置跑通。

### SEO Machine 配置

**1. 创建品牌声音文档：**

在 `seomachine/context/brand-voice.md` 中写入：

```markdown
# 品牌语调

## 品牌简介
[一句话描述你的品牌和产品]

## 目标受众
[谁来读你的内容？职位、痛点、认知水平]

## 语调特征
- 正式/口语化程度：[]
- 技术深度：[] 
- 幽默程度：[]
- 是否用第一人称：[]

## 禁用词汇
[列出你永远不想在内容里看到的词]

## 参考风格
[列出你喜欢的品牌/网站的写作风格]
```

**2. 创建写作样本：**

在 `seomachine/context/writing-examples.md` 中放入 2-3 篇你认为写得好的样本文章（你自己写的或喜欢的）。

**3. 配置目标关键词：**

在 `seomachine/context/target-keywords.md` 中列出 5-10 个你想覆盖的核心方向（不是具体长尾词，是业务方向）。

**4. 配置 WordPress 发布：**

在 `seomachine/wordpress/` 下配置好站点 URL 和应用密码。

### Claude SEO 配置（可选，增强数据）

如有条件，配置 Google API 和 Moz API，让后续审计拿到真实数据。

## 第一阶段：建立内容战略（SuperSEO 主导）

新站最大风险是**方向选错**——写了半年发现选的话题太宽或太难，一篇文章都没排进去。

### 步骤 1：确定第一个话题集群

```
/superseo:topic-cluster-planning
```

输入你的核心业务方向（比如"远程团队协作"）。Agent 会输出：

- **Hub（支柱页）**：一篇覆盖该话题全貌的长文（3000-5000字）
- **Spoke（支撑文章）**：8-12篇聚焦子话题的文章，每篇 1500-2500字
- **内链架构**：每篇 Spoke 如何链接回 Hub，Spoke 之间如何交叉链接
- **发布顺序**：先发哪篇、后发哪篇

**新站建议：** 第一个集群选你业务方向里**最窄但搜索量尚可**的切角。不要一上来就冲着最大最宽的词去——你没有权威度跟老站拼那个。

### 步骤 2：逐个关键词深潜

```
/superseo:keyword-deep-dive
```

对集群里的每个子话题做深潜。Agent 会输出：

- **搜索意图分类**：这个词的用户到底想看到什么内容类型
- **竞争难度评估**：前10都是什么站（政府站、大品牌站、还是小站）？小站多意味着你有机会
- **零点击风险评估**：这个词的 SERP 有没有 Featured Snippet / AI Overview / Knowledge Panel 吃掉了大部分点击
- **90天排名计划**：务实的时间线

**新站策略：** 优先选择满足以下条件的关键词：
1. 前10里有至少2-3个小站或专业博客（说明不是巨头垄断）
2. 搜索量在 100-1000 之间（避开大词，也避开没量的词）
3. 信息型意图为主（新站打商业型/交易型很难）

### 步骤 3：为每篇文章生成内容简报

```
/superseo:content-brief
```

对确定要写的每篇文章，生成一份简报。简报包含：
- 建议的 H2 结构
- 必须覆盖的子话题和角度
- 建议字数
- 竞品覆盖了但你可以做得更好的点

**产出：** 一份清楚的内容日历——接下来 2-3 个月写什么、按什么顺序写。

## 第二阶段：生产第一批内容（SEO Machine 主导）

新站第一批内容宁缺毋滥。**10 篇高质量文章 远好于 50 篇 AI 水文。**

### 步骤 4：研究当前选题

```
/research [主题]
```

在做每篇文章前先研究，输出：
- 前10竞品的内容长度、结构和标题模式
- 你的内容缺口——竞品写了但你没打算写的点
- 搜索意图的细微差别

### 步骤 5：写文章

```
/write [主题]
```

SEO Machine 自动触发四个代理优化：

**写作时的注意事项（新站专属）：**

- **首发文章尽量 2000-3000 字以上**。新站需要向 Google 证明你有深度，短文章很难建立信任
- **开头 200 字直接给答案**。不要在开头讲一堆背景铺垫，用户搜这个词想解决什么，第一段就给他
- **用第一手经验和数据**。新站没有域名权威，只能靠"内容里有别人没有的东西"来打差异——你自己的数据、案例、经验、观点
- **写你真正懂的领域**。新站不要为了 SEO 去写你不懂的领域，E-E-A-T 里 Experience（亲身经验）越来越重要

### 步骤 6：发布前最终优化

```
/optimize [文件路径]
```

确认以下新站特别容易忽略的点：

- **URL 结构清晰**：`domain.com/topic/subtopic/` 而不是 `domain.com/?p=123`
- **每篇文章有唯一且准确的 Title Tag**
- **Meta Description 写的是人话**，不是关键词堆砌
- **图片有文件名**：`remote-team-communication-tools.jpg` 而不是 `IMG_0421.jpg`
- **图片有 Alt Text**
- **内链已加好**：每篇 Spoke 至少链接到 Hub 一篇、其它 Spoke 一篇

### 步骤 7：发布

```
/publish-draft [文件路径]
```

发布后立即做的事：

1. **去 GSC 提交 URL 请求索引**（手动或用 `/seo google inspect` 一次可以批量提交）
2. **检查页面是否能正常访问**：无 404、无重定向链、canonical 指向自身
3. **提交 XML Sitemap**（如果还没做，用 `/seo sitemap` 生成）

## 第三阶段：建立技术基础（Claude SEO 主导）

新站上线的技术配置直接决定后续一切努力的效果。

### 步骤 8：全站技术审计

```
/seo technical [网址]
```

9 个维度的检查，新站重点看：

| 维度 | 新站重点关注 |
|------|-------------|
| **爬取能力** | `robots.txt` 没有误屏蔽任何内容；确认 Googlebot 能访问所有重要页面 |
| **索引能力** | 确保没有任何页面被意外 `noindex`；确认所有页面 `canonical` 指向正确的 URL |
| **安全性** | HTTPS 全站强制；SSL 证书有效。新站没 HTTPS 基本等于白做 |
| **URL 结构** | 层级清晰、可读、含关键词。避免动态参数 (`?id=`, `&session=`) |
| **移动适配** | 移动端可用。Google 用移动优先索引 |
| **Core Web Vitals** | LCP < 2.5s。新站通常页面少、服务器配置影响不大，主要是图片和字体优化 |
| **结构化数据** | 至少部署 `Organization` 和 `WebSite` Schema（用下面步骤9做） |

### 步骤 9：部署基础 Schema

```
/seo schema [网址]
```

新站至少需要两块 Schema：

- **Organization**：告诉 Google 你是谁，什么品牌名，什么 Logo
- **WebSite** + **SiteNavigationElement**：告诉 Google 你的网站结构和搜索框

如果你有博客，额外加 `BlogPosting` Schema 在每篇文章上。

### 步骤 10：生成并提交 XML Sitemap

```
/seo sitemap [网址]
```

对收录来说，sitemap 是新站最重要的文件之一——新站没有外链，Google 基本靠 sitemap 发现新页面。

**新站 Sitemap 最佳实践：**

- 只收录你想被索引的页面（不要把后台页、感谢页也放进去）
- 只收录 200 状态的页面
- 如果有图片和视频，在 sitemap 里声明 `<image:image>` 和 `<video:video>`
- 提交到 GSC 后每隔几天检查一次 sitemap 的"已发现"vs"已收录"比例

### 步骤 11：建立 SEO 漂移基线

```
/seo drift baseline [网址]
```

新站上线后立即建立基线。以后每次改版、更新设计、换主题，都能用 `compare` 对比看有没有意外改坏了什么。

## 第四阶段：内容质检与迭代（SuperSEO 主导）

第一批内容发布后 2-4 周，开始质检和迭代。

### 步骤 12：逐篇页面审计

```
/superseo:page-audit
```

对已发布的核心页面逐一审计，7个维度打分：
1. 内容质量（深度、原创性、实用性）
2. 技术 SEO（meta、headings、canonical）
3. E-E-A-T 信号（作者信息、引用来源、举证）
4. 内容结构（H2/H3 层级、信息层级）
5. 内部链接（是否有相关文章链接、锚文本是否准确）
6. Schema 标记（是否部署了正确的结构化数据）
7. 竞争定位（对标前3，你哪些方面更强、哪些方面不足）

### 步骤 13：E-E-A-T 专项审计

```
/superseo:eeat-audit
```

新站没有品牌名气，E-E-A-T 是唯一的竞争力来源。Agent 会在四个维度逐项评分 1-10，并给出具体可操作的改进建议：

- **Experience（经验）**：有个人/团队的实际经验佐证吗？有没有案例、数据、截图、视频来证明"你做过"而不是"你搜到过"
- **Expertise（专业知识）**：作者在这个领域的资质是什么？有没有引用行业公认的资源？内容深度是否达到专业水平
- **Authoritativeness（权威性）**：有没有其他权威来源引用你（这个新站暂时没有，但可以做到在内容中引用权威来源建立关联）
- **Trustworthiness（可信度）**：联系信息、隐私政策、关于页是否完整？信息是否准确、有据可查

### 步骤 14：根据审计结果迭代

```
/superseo:improve-content
```

对评分偏低的页面，用这个命令重写薄弱段落。

或者回到 SEO Machine 用 `/rewrite [主题]` 做更大范围的改进。

## 第五阶段：第一批外链建设（SuperSEO + Claude SEO）

通常在新站上线 2-3 个月、有了 10-15 篇内容后开始。

### 步骤 15：确定外链策略阶段

```
/superseo:linkbuilding
```

Agent 会通过可见信号判断你的网站处在**基础期（Foundation）**，推荐适合基础期的战术：
- 行业目录提交
- 供应商/合作伙伴页面要链接
- 客座博客（在相关小站上写嘉宾文章）
- 资源列表页面（"XX行业最佳资源"类文章里申请加入）

### 步骤 16：监控外链增长

```
/seo backlinks [网址]
```

定期（每月）用 Claude SEO 检查外链增长：
- 新获得的链接质量
- 是否有可疑链接需要 disavow
- 竞品在哪些域拿到了链接，你能不能也去拿

## 新站发布前检查清单

在上线前逐项确认：

### 技术基础
- [ ] HTTPS 全站强制
- [ ] robots.txt 正确，未误屏蔽
- [ ] XML Sitemap 已生成并提交 GSC
- [ ] GSC 已验证所有权
- [ ] 移动端可用
- [ ] Core Web Vitals 及格（LCP < 2.5s）
- [ ] 无404、无重定向链

### Schema
- [ ] Organization Schema 已部署（含 Logo）
- [ ] WebSite Schema 已部署
- [ ] 文章页 BlogPosting Schema 已部署

### 内容
- [ ] 至少 8-12 篇高质量文章已就绪
- [ ] Hub 支柱页已完成
- [ ] 内链结构完整（每篇都有链出和链入）
- [ ] 标题标签和描述都是人写的、独一无二的
- [ ] 品牌声音文档已配置

### 页面体验
- [ ] 有关于页（About Page）——新站 E-E-A-T 的基础
- [ ] 有联系页（Contact Page）
- [ ] 有隐私政策页
- [ ] 文章有清晰的作者署名和简介

## 第1-6个月节奏规划

| 月份 | 核心工作 | 使用工具 |
|------|---------|---------|
| **第1月** | 确定内容战略；写完第一批 8-12 篇 + Hub 页；配置技术基础 | SuperSEO（战略）+ SEO Machine（生产）+ Claude SEO（技术） |
| **第2月** | 第一批内容发布；逐篇质检迭代；补 E-E-A-T 短板 | SEO Machine（发布）+ SuperSEO（质检） |
| **第3月** | 启动外链建设；继续按内容日历写第二批文章 | SuperSEO（外链规划）+ SEO Machine（写新内容） |
| **第4月** | 全站技术审计；对比漂移基线；分析 GSC 前30天数据找机会 | Claude SEO（审计漂移）+ SEO Machine（写机会内容） |
| **第5月** | 根据 GSC 数据识别"低挂果实"——排在 5-15 名的词集中优化 | SuperSEO（semantic-gap + snippet优化）+ SEO Machine（rewrite） |
| **第6月** | 全站审计 + 外链审计；决定是否开启第二个话题集群 | Claude SEO（全站体检）+ SuperSEO（新集群规划） |

## 常见新站错误

**错误1：第一个月铺50篇文章。**
Google 不认识你的时候，发再多内容也排不上去。10 篇高质的远好于 50 篇水的。

**错误2：追大词。**
新站追大词（比如"CRM software"、"project management"）等于用自行车追跑车。先在小词上建立存在感。

**错误3：发完不管。**
新站内容发布后不是终点。第2-4周是关键观察期——哪篇开始有展示没点击？哪篇根本没被收录？

**错误4：内链马虎。**
没有外链的情况下，内链是 Google 理解你网站结构的唯一线索。每篇文章必须有明确的链接上下文。

**错误5：忽略 E-E-A-T 信号。**
没有作者页、没有关于页、没有引用来源的文章，新站发出去 Google 不敢排。
