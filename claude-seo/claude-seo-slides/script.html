0:35 - 2:15
Google SEO APIs
Screen: Show terminal with Claude Code open

The biggest addition is direct Google API integration. The first version of Claude SEO was smart but blind — it could analyze your HTML, but it couldn't see what Google actually sees. Now it can.

Claude SEO pulls live data from six Google APIs — Search Console, PageSpeed Insights, CrUX, GA4, the Indexing API, and YouTube Data.

$ /seo google psi https://example.com

So here I'm running a PageSpeed check. This isn't just lab data — it pulls real Chrome User Experience data, the actual Core Web Vitals that Google uses for ranking. LCP, INP, CLS — all from real users.

The credential system has four tiers. You can start completely free:

  • Tier 0 — free API key — gives you PageSpeed, CrUX, CrUX History, YouTube, and NLP
  • Tier 1 — add OAuth — unlocks Search Console, URL Inspection, and the Indexing API
  • Tier 2 — add GA4 — organic traffic reports, top landing pages, device breakdown
  • Tier 3 — add Google Ads — Keyword Planner data

One API key, five minutes of setup, and you already have live performance data. No paid tools needed.

$ /seo google report full

And all of this feeds into professional PDF reports. A4 format, charts, color-coded scores, table of contents — something you can actually hand to a client or a boss.

Show generated PDF — flip through a few pages, pause on charts

New in v1.7.2 — Excel export.

$ /seo google report full --format all

That gives you PDF, HTML, and an Excel workbook all at once. The spreadsheet has separate sheets for queries, pages, indexation — with filters, frozen headers, auto-width columns. Clients love data they can sort.

2:15 - 3:30
DataForSEO — Live SEO Data
Screen: Terminal

The second big feature is the DataForSEO extension. This gives Claude SEO access to live SERP data, keyword research, backlink profiles, and a lot more.

$ /seo dataforseo serp best coffee shops new york

So here I'm pulling a live Google SERP. Not cached, not estimated — this is what's actually ranking right now. You see positions, URLs, featured snippets, People Also Ask, local pack results — everything.

$ /seo dataforseo keywords seo audit tool

Keyword research — search volume, difficulty score, CPC, competition level. This is the same data you'd get from paid tools like Ahrefs or SEMrush, but you're running it from your terminal inside Claude Code.

$ /seo dataforseo backlinks example.com

And backlink analysis. Referring domains, anchor text distribution, domain rating. We also added a dedicated /seo backlinks command in v1.7.2 that wraps this data with a 7-section analysis framework — toxic link detection with 30 spam patterns, competitor gap analysis, and a health score from 0 to 100.

DataForSEO has 22 commands across 9 API modules. SERP, keywords, backlinks, on-page analysis, domain analytics, business data, content analysis, and AI optimization. It's a lot.

And installing it is one script:

$ ./extensions/dataforseo/install.sh

It asks for your API credentials, configures the MCP server, and you're done. Which brings me to the third thing I want to show you.

3:30 - 4:15
The Extension System
Screen: Show the extensions/ folder in editor, then the before/after slide

This is the part I'm most proud of because it's how Claude SEO scales without getting bloated.

The core skill — 17 sub-skills, 12 subagents — works with zero API keys. You install it, run /seo audit, and 12 AI agents analyze your site in parallel. Technical SEO, content quality, schema, GEO, local SEO — all of that runs on Claude's own intelligence, no external APIs.

But when you want deeper data, you plug in an extension. Right now there are three:

  • DataForSEO — 22 commands: live SERPs, keyword volume, backlinks, content analysis
  • Firecrawl — full-site crawling with JavaScript rendering, 500 free credits/month
  • Banana — AI image generation via Google Gemini for OG images, hero graphics, product shots

Each extension is self-contained. One install script, one uninstall script. It copies the skill files, configures the MCP server, done. And the audit orchestrator detects what's installed — if you have DataForSEO, it automatically spawns the DataForSEO agent during audits. If you have Firecrawl, it maps your site first. The more you plug in, the deeper your audits get.

And building new extensions is easy. Same pattern, same structure. If you want to add Ahrefs, SEMrush, or any MCP server — you follow the template.
