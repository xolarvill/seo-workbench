technical deep-dive

DataForSEO Cost Guardrails

DataForSEO calls accumulate cost fast when multiple subagents run in parallel. v1.9.0 adds a cost estimation and approval workflow in scripts/dataforseo_costs.py.

The Bypass Chain (Closed)
Unknown endpoints defaulted to $0.05 per call, bypassing the approval gate by using an undocumented endpoint name
The reset command cleared cumulative spend with no confirmation, letting a misconfigured agent wipe the budget ledger and restart a spending cycle
# Before (bypass possible) unit_cost = COST_TABLE.get(endpoint, 0.05) # After (fixed) unit_cost = COST_TABLE.get(endpoint) if unit_cost is None: return {"status": "needs_approval", "approval_reason": "unknown_endpoint"} # Reset: now requires --confirm def cmd_reset(args): if not args.confirm: return {"status": "blocked", "message": "Reset requires --confirm flag."}
Concurrent Safety (fcntl)

The cost ledger is a JSON file read and written by multiple parallel subagents. v1.9.0 adds fcntl.flock(): shared lock for reads, exclusive lock for writes. Windows falls back gracefully.

Approval Workflow
Endpoint: merchant_competitors
Estimated: $0.18  |  Budget: $5.00
Proceed? [y/N] ← new prompt
Attribution: Cost guardrail concept from Matej Marjanovic. Applied globally across all DataForSEO calls during integration.

v1.9.1 Security Priorities

H1: DNS Rebinding Gap HIGH

validate_url() checks IPs at parse time but not at request time. Fix: re-validate resolved IPs at the socket layer.

H2: Install Script Injection HIGH

install.sh and install.ps1 use unquoted variables. A path with spaces could inject commands. Fix: quote all variable expansions.

M1: OAuth File Permissions MEDIUM

OAuth token files written world-readable on permissive umask systems. Fix: chmod 600 immediately after writing.
