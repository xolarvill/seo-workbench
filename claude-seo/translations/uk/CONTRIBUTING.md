# Внесок у claude-seo

Дякуємо за інтерес до участі! Ось як долучитися.

## Повідомлення про помилки

Відкрийте [GitHub Issue](https://github.com/AgriciDaniel/claude-seo/issues) із зазначенням:

- Вашої ОС та версії Python
- Повного виводу помилки (скопіюйте з терміналу)
- Команди або кроку, який не вдався
- URL, який ви аналізували (якщо застосовно)

## Пропозиції функцій

Використовуйте [GitHub Discussions](https://github.com/AgriciDaniel/claude-seo/discussions) для ідей та питань.

## Pull Requests

1. Зробіть форк репозиторію
2. Створіть гілку функції (`git checkout -b feature/my-feature`)
3. Внесіть зміни
4. Протестуйте на реальному URL перед відправкою
5. Подайте PR з чітким описом що змінено та чому

### Налаштування середовища розробки

```bash
git clone https://github.com/YOUR_USERNAME/claude-seo.git
cd claude-seo
bash install.sh
```

### Настанови

- Усі Python-скрипти повинні виводити JSON для парсингу Claude Code
- Shell-скрипти повинні використовувати `set -euo pipefail` для безпеки
- Файли SKILL.md мають бути не довшими за 500 рядків
- Довідкові файли мають бути цілеспрямованими та не довшими за 200 рядків
- Використовуйте kebab-case для назв усіх каталогів та файлів
- Мінімізуйте залежності

### Стиль коду

- Python: Дотримуйтесь конвенцій PEP 8. Використовуйте `ruff check` або `flake8` для лінтингу перед відправкою
- Shell: Використовуйте `set -euo pipefail` та лапки для всіх змінних
- Markdown: Тримайте рядки до 120 символів де це можливо
