Executive Summary

Version 1.8.0 transforms the /seo backlinks command from a paid-only feature into a multi-tier free-to-premium backlink analysis system. Users can now get actionable backlink intelligence with zero cost and zero configuration, with progressively richer data as they add free API keys.

Key achievement: Free sources combined can capture 60–70% of actionable backlink intelligence — the highest-authority links most likely appear in free tool samples. For sites with fewer than 500 backlinks, free sources capture 50%+ of the meaningful profile.

Architecture

3-Tier Cascade

Tier 0 — Always Available ZERO CONFIG
Common Crawl Web Graph: Domain-level PageRank, harmonic centrality from the latest quarterly release (134M+ domains). Streams data via incremental decompression — no full download needed.
Verification Crawler: Fetches source pages, parses HTML, confirms if backlinks exist. Detects JS-rendered pages. Polite crawling (1 req/sec per domain).
Tier 1–2 — Free Signup FREE
Moz API: Domain Authority, Page Authority, Spam Score, referring domains, anchor text. 2,500 rows/month, 1 req/10 sec. JSON-RPC 2.0 at api.moz.com.
Bing Webmaster Tools: Inbound links with anchor text + unique competitor comparison feature. Free for verified sites.
Tier 3 — Premium PAID
DataForSEO: 35+ trillion links, real-time updates, toxic scoring, competitor gap analysis. Existing integration completely untouched — all v1.7.2 DataForSEO workflows still work identically.

Confidence Weighting

When merging data from multiple sources, each metric is weighted by source reliability:

SourceConfidenceCoverage vs CommercialUpdate Frequency
DataForSEO1.00~90%+Real-time
Verification Crawler0.95N/A (binary check)On-demand
Moz API0.85~70% for DA/PA~3 days
Bing Webmaster0.70~15% (Bing index)Near real-time
Common Crawl0.50~25–40% domainsQuarterly
Data sufficiency gate: If fewer than 4 of 7 scoring factors have data, the system reports "INSUFFICIENT DATA" instead of a misleading numeric score. A score of 25/100 with 1 data source would imply "bad site" when the reality is "not enough data."

Data Accuracy Safeguards

Three layers of protection ensure reports are accurate before reaching the user:

Layer 1: Script-Level Detection

ProtectionWhat It Catches
JS-rendering detectionInstagram, Facebook, SPAs reported as unverifiable_js instead of false link_removed
Suspicious H1 flaggingCounter/stat numbers (e.g., "67") flagged in h1_suspicious, not reported as headings
@graph schema flatteningValid JSON-LD using @graph wrapper correctly parsed as individual @type entries
CC in_crawl vs in_rankingsDistinguishes "not crawled" from "crawled but below ranking threshold"
Subdomain matchingshop.example.com correctly matches target example.com

Layer 2: Automated Validator

validate_backlink_report.py runs 6 programmatic checks on collected data:

CheckCatches
Schema claimsUnflattened @graph, missing @type, deprecated types
Verification resultsSocial media pages falsely marked link_removed, summary/detail mismatches
H1 claimsAll H1s suspicious (no real heading)
CC interpretation"Not crawled" misinterpreted as "low authority"
Reciprocal linksBidirectional link patterns (A→B and B→A)
Health scoreNumeric score with insufficient data (<4/7 factors)

Layer 3: Agent Instruction Checklist

The skill and agent include a mandatory pre-delivery review with 11 checks covering source labels, inference vs fact, platform detection, and cross-check consistency.

Security

ProtectionImplementation
